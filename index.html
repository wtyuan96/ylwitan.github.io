Dijkstra算法（优先队列）

发表于 2019-12-19 | 分类于 算法

Dijkstra算法只能求取边的权重为非负的图的最短路径，而Bellman-Ford算法可以求取边的权重为负的图的最短路径（但Bellman-Ford算法在图中存在负环的情况下，最短路径是不存在的）。

算法原理

Dijkstra算法本质上是一种贪心算法，1959年，Edsger Dijkstra提出了该算法用于解决单源最短路径问题，即在给定每条边的长度$\mathcal{l}$，求取源节点s到其它所有节点的最短路径。

Dijkstra算法维护一个节点集合S，S中的节点到源点的最短距离d已经确定了，即“Explored”。

初始化时，S={s},d(v)=0，接下来，对于每一个节点，选出到集合S的距离最近的节点$v^*$，即最小化下面这个问题：

我们将$v^$加入到节点S，并定义$d(v^)=d’(v^*)$。

Checkinstall是将源码转换成deb或者rpm包的工具，操作简单。在编译完成后，首先修改一下checkinstall的配置文件checkinstallrc，可以通过whereis checkinstallrc查找此文件的位置，打开文件修改BACKUP变量为0（默认为1），这个变量控制在checkinstall过程中是否保存备份文件，由于Opencv编译生成的文件太大，保存备份文件会很耗时，所以选择不保存备份文件。
修改完checkinstall的配置文件后，直接运行sudo checkinstall即可，会在build目录下生成一个.deb包，此时Opencv已经安装上了，以后如果需要删除Opencv，直接运行：sudo dpkg -r build，而如果要安装直接运行sudo dpkg -i build_xxx.deb即可。

Windows+Ubuntu双系统，Ubuntu系统扩容

发表于 2019-05-18 | 更新于 2019-06-26 | 分类于 Linux

Windows 10下安装了Ubuntu双系统，刚开始给Ubuntu分配了100G的空间，但是用了几个月发现100G开始不太够用了，遂考虑给Ubuntu加50G空间，记录过程如下。
参考原文链接：How to Resize Your Ubuntu Partitions，在此翻译总结一下主要流程。
无论是给扩大还是缩小Ubuntu系统容量，抑或是更改系统分区，都必须在Ubuntu系统不运行的情况下。所以你需要制作一个Ubuntu的启动盘来更改系统分区。
目前Ubuntu的启动盘都包含GParted分区编辑软件（Ubuntu18.04确定含有）。

制作Ubuntu启动盘

制作启动盘的软件有很多，这里采用UltraISO，首先从Ubuntu官网下载Ubuntu18.04的镜像文件，然后找一个空的U盘（或里面内容没啥用的U盘），然后用UltraISO制作启动盘，制作系统盘的教程网上很多，在此不详述。

设置U盘启动并使用GParted进行分区调整

首先进BIOS设置从U盘启动，接下来会进入Ubuntu系统。
在Dash中输入GParted，打开GParted软件。

首先需要关闭交换分区（swap partition），右击，点击Swap off。

右击某个分区，然后点击Resize/Move来更改分区。

最简单的修改分区大小的方式就是直接点击并拖动滚动条，当然也可以直接输入准确的数字来改变大小。

你对于分区的改动不会立即生效。你做的每一个改动都会进入一个队列，并在GParted窗口的下面展示出来。

一旦你缩小了一个分区，你可以用未分配的空间新建一个分区，直接右击未分配的空间，然后选择New，GParted会指导你创建新分区。

如果一个分区与未分配的空间临近，可以直接右击这个分区，然后选择Resize/Move，将未分配的空间分配进这个分区。
如果更改了Ubuntu的开始扇区（或者更改了Ubuntu的分区数量）,Ubuntu可以不能boot，可以通过重新安装Grub2来解决这个问题，参见修复Grub2引导

最后需要点击绿色对号图标来使你的改动生效。

Tips:

  1. 我们通常扩容的是Home分区或者/分区（这两个分区也可以是一个分区），所以如果是双系统的话，在安装Ubuntu系统的时候，把Home分区或者/分区分配在靠近Windows分区的位置，这样以后更改分区可以直接缩小Windows分区，然后并入Home分区或者/分区，这样GParted在调整分区时会免去复制文件的过程，更节省时间。
  2. 如果单纯缩小某个分区，然后扩大另一个分区，而不改变分区位置和数量的话，不会出现无法boot的现象。
